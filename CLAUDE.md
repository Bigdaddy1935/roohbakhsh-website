# راهنمای پروژه برای Claude — آکادمی روح‌بخش

> این فایل را قبل از هر کاری بخوان. قوانین اینجا روی همه‌ی اپ‌ها اعمال می‌شود.
> هر اپ ممکن است `CLAUDE.md` خودش را هم داشته باشد که قوانین خاص‌تری اضافه می‌کند.

## پروژه چیست
وب‌سایت آکادمی بین‌المللی اسلامی روح‌بخش. چندزبانه (عربی + اردو)، کاملاً RTL، با هویت اسلامی.
ساخت در ۳ فاز: (۱) سایت لندینگ سبک، (۲) CRM + دوره‌ها + بلاگ + SEO، (۳) LMS کامل.

## ساختار مونوریپو
```
apps/web   → سایت اصلی (Next.js, App Router)
apps/cms   → CMS هدلس (Payload/Strapi) — مدیریت محتوا
apps/api   → بک‌اند (NestJS) — احراز هویت، پرداخت، LMS، تیکت
packages/shared → تایپ‌های مشترک. منبع حقیقت قرارداد API.
```

## قانون شماره ۱ — قرارداد، مقدم بر کد
- شکل هر داده‌ی بین فرانت و بک، **اول** در `packages/shared` تعریف می‌شود، بعد کد.
- همیشه از تایپ‌های `@roohbakhsh/shared` استفاده کن. تایپ تکراری محلی نساز.
- اگر نیاز به تغییر شکل یک API بود: اول `docs/API-CONTRACT.md` و `packages/shared` را آپدیت کن.

## قانون شماره ۲ — چندزبانگی و RTL
- هر متنی که کاربر می‌بیند باید از نوع `Localized` باشد (`{ ar, ur }`). **هیچ متن hard-code فارسی/انگلیسی در UI ننویس.**
- صفحه‌ها همیشه RTL هستند. در طراحی از منطق logical استفاده کن (`padding-inline-start` نه `padding-left`).
- فونت‌ها: عربی → Almarai، اردو → Amar Nastaleeq.

## قانون شماره ۳ — هویت بصری (پالت رنگ ۶۰-۳۰-۱۰)
از این مقادیر استفاده کن، رنگ دلخواه نساز:
- پس‌زمینه (۶۰٪): سفید شیری `#F8F9FA`
- برند/حاشیه/آیکون (۳۰٪): سبز زمردی `#0CA789`
- دکمه‌های CTA (۱۰٪): نارنجی گرم `#FF9F1C`
- متن و تیتر: سرمه‌ای تیره `#2B2D42`
- دکمه‌ها گوشه‌گرد. کارت دوره: پس‌زمینه سفید خالص + سایه‌ی ملایم + حاشیه‌ی سبز زمردی.

## قانون شماره ۴ — کیفیت کد
- TypeScript strict. از `any` استفاده نکن؛ اگر مجبوری، با کامنت دلیلش را بنویس.
- نام‌گذاری و سبک کد را با کد اطراف هماهنگ کن.
- هر API بک‌اند باید ورودی را با DTO اعتبارسنجی کند و خطا را به شکل `ApiError` برگرداند.
- کامپوننت‌ها کوچک و قابل‌استفاده‌ی مجدد باشند.

## قانون شماره ۵ — گردش کار Git
- روی `main` مستقیم کد نزن. برنچ بساز: `feature/...` یا `fix/...`.
- پیام کامیت کوتاه و واضح. کامیت‌های منطقی و کوچک.
- قبل از تمام‌کردن کار، `pnpm typecheck` و `pnpm lint` باید سبز باشند.

## دستورها
```
pnpm install     # یک‌بار، اول کار
pnpm dev         # کل پروژه را لوکال بالا می‌آورد (web + api)
pnpm typecheck   # بررسی تایپ همه‌ی اپ‌ها
pnpm lint
```

## چیزهایی که نباید بکنی
- تایپ مشترک را در یک اپ کپی نکن؛ از `@roohbakhsh/shared` بیاور.
- متن قابل‌نمایش را hard-code نکن.
- روی `main` مستقیم کامیت نکن.
- بدون به‌روزرسانی قرارداد، شکل API را عوض نکن.
