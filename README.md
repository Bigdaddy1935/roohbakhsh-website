# استارتر آکادمی روح‌بخش

این پوشه نقطه‌ی شروع هماهنگی تیم است. هدفش این است که **سه نفر موازی و بدون معطلی** کار کنند و هیچ‌وقت از هم جدا نشوند.

## این پوشه چه چیزی دارد؟

```
roohbakhsh-starter/
├── README.md                    ← همین فایل
├── CLAUDE.md                    ← مغز پروژه: قوانینی که Claude همه می‌خواند
├── package.json / turbo.json    ← مونوریپو (pnpm + Turborepo)
├── docs/
│   └── API-CONTRACT.md          ← سند جلسه‌ی «قدم ۰». منبع حقیقت APIها.
├── packages/
│   └── shared/                  ← تایپ‌های مشترک (Course, User, AuthResponse, ...)
└── apps/
    ├── web/   (Next.js)         ← سایت — RTL + i18n + اتصال به API. دارای CLAUDE.md
    ├── api/   (NestJS)          ← بک‌اند — Swagger + endpoint نمونه. دارای CLAUDE.md
    └── cms/                     ← placeholder (ابزارش در قدم ۰ تعیین می‌شود)
```

> هر اپ یک `CLAUDE.md` خودش دارد. وقتی برنامه‌نویس داخل آن اپ با Claude کار می‌کند،
> Claude اول قوانین ریشه بعد قوانین همان اپ را می‌خواند — کد یکدست و طبق استاندارد در می‌آید.

## راه‌اندازی (یک‌بار)

```
nvm use            # Node 20
pnpm install       # نصب همه‌ی اپ‌ها
cp apps/web/.env.local.example apps/web/.env.local
pnpm dev           # web → :3000/ar   |   api → :3001/api/docs
```

> این اسکلت با تایپ‌چک تأیید شده. اپ‌های web/api حداقلی ولی واقعی‌اند؛ بعد از `pnpm install`
> با `pnpm dev` بالا می‌آیند. (نصب پکیج‌ها در این محیط اجرا نشده — تیم خودش `pnpm install` می‌زند.)

## تقسیم کار تیم

| نفر | مسئولیت | اپ |
|-----|---------|-----|
| فرانت‌کار ۱ | CMS (مدل محتوا، پنل ادمین، API محتوا) | `apps/cms` |
| فرانت‌کار ۲ | سایت (UI، صفحات، فرم‌ها) | `apps/web` |
| بک‌اندکار | بک‌اند + دیتابیس | `apps/api` |

> `apps/*` هنوز ساخته نشده‌اند. بعد از جلسه‌ی قدم ۰ ساخته می‌شوند. این استارتر فقط پایه‌ی هماهنگی (قرارداد + تایپ مشترک) است.

## ترتیب کار

1. **جلسه‌ی قدم ۰ (نیم‌روز، هر سه نفر + ناظر):**
   `docs/API-CONTRACT.md` را با هم بخوانید، کامل و تأیید کنید. مرز CMS و NestJS را قطعی کنید.

2. **بک‌اندکار اول این‌ها را می‌سازد و push می‌کند:**
   - تایپ‌های `packages/shared` را نهایی کند.
   - اسکلت endpointها در NestJS (حتی با داده‌ی ساختگی) + فعال‌کردن Swagger روی `/api/docs`.

3. **از این لحظه هر سه موازی کار می‌کنند:**
   - بک‌اندکار → منطق واقعی + دیتابیس.
   - فرانت‌کار سایت → UI با تایپ مشترک و داده‌ی mock، بعد اتصال به API واقعی.
   - فرانت‌کار CMS → کالکشن‌ها و پنل محتوا.

## قوانین طلایی هماهنگی

1. هیچ کدی مستقیم روی `main` نرود. هر کار → برنچ → PR → ریویو → merge.
2. هر تغییر API: **اول** `API-CONTRACT.md` و `packages/shared`، **بعد** کد.
3. هر متن قابل‌نمایش باید `Localized` باشد (عربی + اردو). هیچ متن hard-code فارسی/انگلیسی در UI نباشد.
4. صبح هر روز `git pull`. آخر هر کار `git push` + PR.

## چطور تایپ مشترک استفاده می‌شود؟

در هر اپ:
```ts
import { Course, AuthResponse, Paginated } from "@roohbakhsh/shared";
```
اگر بک‌اندکار شکل `Course` را عوض کند، ادیتور فرانت‌کار **فوراً خطا می‌دهد**. این تضمین می‌کند فرانت و بک همیشه همگام بمانند.
