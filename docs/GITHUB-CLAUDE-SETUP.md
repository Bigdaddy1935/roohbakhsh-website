# راه‌اندازی ریویوی خودکار Claude روی GitHub

هدف: هر PR که بچه‌ها می‌زنند، خودکار توسط Claude ریویو شود و تو (ناظر) بدون خواندن تک‌تک خطوط، نظارت داشته باشی.

## دو ورک‌فلویی که ساخته شده

| فایل | کِی اجرا می‌شود | چی‌کار می‌کند |
|------|----------------|---------------|
| `.github/workflows/claude-code-review.yml` | باز شدن هر PR + هر push جدید | کل diff را طبق `CLAUDE.md` ریویو و کامنت inline می‌گذارد |
| `.github/workflows/claude.yml` | منشن `@claude` در کامنت/issue | جواب سؤال می‌دهد یا باگ را اصلاح و push می‌کند |

## مراحل راه‌اندازی (یک‌بار، توسط تو)

### ۱) کلید API بگیر
- برو به **console.anthropic.com** → بخش API Keys → یک کلید بساز.
- این **جدا از اشتراک Max** است و مصرفی است (به‌ازای توکن). هزینه‌ی ریویوی PRها معمولاً ناچیز است.
- یک سقف بودجه (Billing limit) روی کنسول بگذار تا خرج کنترل‌شده بماند.

### ۲) کلید را به‌عنوان Secret در ریپو ثبت کن
در صفحه‌ی ریپو روی GitHub:
`Settings → Secrets and variables → Actions → New repository secret`
- **Name:** `ANTHROPIC_API_KEY`
- **Secret:** کلیدی که از کنسول گرفتی

> اگر چند ریپو دارید، می‌توانی به‌جای هر ریپو، یک **Organization secret** بسازی تا برای همه مشترک باشد.

### ۳) اپ گیت‌هاب Claude را نصب کن
ساده‌ترین راه: در ترمینال، داخل پروژه `claude` را باز کن و بزن:
```
/install-github-app
```
خودش تو را قدم‌به‌قدم می‌برد (نصب اپ روی ریپو + ثبت secret). اگر این را زدی، عملاً مرحله‌ی ۲ هم انجام می‌شود.

اگر دستی می‌خواهی: **github.com/apps/claude** → Install → ریپوی پروژه را انتخاب کن.

### ۴) تست کن
- یک برنچ بساز، یک تغییر کوچک بزن، PR باز کن.
- چند ثانیه بعد، تب **Actions** را ببین؛ job «Claude PR Review» باید اجرا شود و زیر PR کامنت بگذارد.
- زیر PR بنویس `@claude این کد را توضیح بده` و ببین جواب می‌دهد.

## محافظت از شاخه‌ی main (توصیه‌ی مهم)

تا کسی نتواند بدون ریویو روی `main` merge کند:
`Settings → Branches → Add branch ruleset`
- روی `main`: **Require a pull request before merging** را روشن کن.
- **Require approvals: 1** (تأیید تو).
- اختیاری: ریویوی Claude را به‌عنوان status check اجباری کن.

## نکته درباره‌ی هزینه و کنترل
- ریویوی خودکار = هزینه‌ی API مصرفی. با تنظیم `types: [opened, synchronize]` فقط روی PR اجرا می‌شود، نه هر کامیت.
- اگر خواستی فقط روی PRهای آماده اجرا شود (نه draft)، می‌توان شرط `if` اضافه کرد.
- `concurrency` در ورک‌فلو، pushهای پشت‌سرهم را لغو می‌کند تا هزینه‌ی تکراری ندهی.
