# راهنمای کامل پروژه — آکادمی بین‌المللی روح‌بخش

این سند را قبل از هر کاری، کامل بخوان. هدفش این است که بدانی **پروژه چیست، چطور چیده شده، و چطور شروع به کار کنی** — بدون اینکه چیزی مبهم بماند.

---

## ۱) پروژه چیست؟

وب‌سایت آکادمی بین‌المللی اسلامی **روح‌بخش**. سایتی **چندزبانه** (عربی + اردو)، کاملاً **راست‌به‌چپ (RTL)**، با هویت بصری اسلامی. در ۳ فاز ساخته می‌شود:

- **فاز ۱:** سایت لندینگ سبک و سریع + چندزبانه + فرم ثبت‌نام/ایمیل + آپلود ویدیو/عکس + تیکتینگ ساده
- **فاز ۲:** CRM + دوره‌ها + بلاگ + بهینه‌سازی SEO
- **فاز ۳:** سامانه‌ی آموزش (LMS) کامل — پنل کاربر، پنل مدرس/مدیر، گواهی، جلسات آنلاین، درگاه پرداخت ارزی

---

## ۲) معماری: سه اپ که با هم یک پروژه‌اند

این یک **مونوریپو** است: یک ریپوی واحد که چند اپ داخلش است. همه‌ی کد در همین یک ریپوست.

```
roohbakhsh-website/
├── apps/
│   ├── web/      → سایت اصلی (Next.js) — چیزی که بازدیدکننده می‌بیند
│   ├── cms/      → پنل مدیریت محتوا (Headless CMS) — جای ساختن دوره/بلاگ/استاد
│   └── api/      → بک‌اند (NestJS) — احراز هویت، پرداخت، LMS، تیکت
├── packages/
│   └── shared/   → تایپ‌های مشترک TypeScript (قلب پروژه — پایین توضیح داده شده)
├── docs/         → همین مستندات و قرارداد API
├── CLAUDE.md     → قوانین پروژه که Claude موقع کدزدن می‌خواند
└── package.json  → ریشه‌ی مونوریپو (pnpm + Turborepo)
```

### هر اپ مال کیست و چه‌کار می‌کند؟

| اپ | مسئول | چه‌کار می‌کند |
|-----|--------|----------------|
| `apps/web` | فرانت‌کار | همه‌ی صفحات سایت: لندینگ، دوره‌ها، فرم‌ها، داشبورد کاربری |
| `apps/cms` | فرانت‌کار | مدل محتوا و پنل ادمین برای تیم محتوا. **API محتوا** را به سایت می‌دهد |
| `apps/api` | بک‌اندکار | منطق کسب‌وکار: حساب کاربری، پرداخت، LMS، گواهی، تیکت |

> **مرز مهم:** محتوای دوره/استاد/بلاگ از **CMS** می‌آید. حساب کاربری/پرداخت/LMS از **NestJS** می‌آید.
> سایت (`web`) به **هر دو** وصل می‌شود.

## ۳) استک فناوری (نسخه‌های فعلی)

| بخش | فناوری |
|------|---------|
| سایت | **Next.js 16** (App Router) + **React 19** + **next-intl 4** (چندزبانگی) |

## ۴) قلب پروژه: پکیج مشترک (`packages/shared`) و «قرارداد»

این مهم‌ترین مفهوم پروژه است. خوب بخوان.

### مشکلی که حل می‌کند
وقتی فرانت و بک جدا کار می‌کنند، همیشه این سؤال پیش می‌آید: «این API دقیقاً چه داده‌ای برمی‌گرداند؟ اسم فیلدش `title` بود یا `name`؟» اگر این هماهنگ نباشد، باگ‌های زیادی ساخته می‌شود.

### راه‌حل: یک منبع حقیقت مشترک
شکل همه‌ی داده‌هایی که بین فرانت و بک رد و بدل می‌شود، **یک‌بار** در `packages/shared` به‌صورت تایپ TypeScript تعریف می‌شود. مثال:

```ts
// packages/shared/src/course.ts
export interface CourseListItem {
  id: string;
  slug: string;
  title: Localized;        // { ar: "...", ur: "..." }
  price: Money | null;     // null یعنی رایگان
  instructor: InstructorSummary;
  hasFreePreview: boolean;
}
```

حالا **هم بک‌اند، هم سایت، هم CMS** همین تایپ را import می‌کنند:

```ts
import { CourseListItem, Paginated } from "@roohbakhsh/shared";
```

### چرا جادویی است؟
اگر بک‌اندکار اسم یک فیلد را عوض کند (مثلاً `title` → `heading`)، **ادیتورِ فرانت‌کار فوراً قرمز می‌شود و خطا می‌دهد** — قبل از اینکه حتی برنامه را اجرا کند. یعنی فرانت و بک هیچ‌وقت از هم جدا نمی‌شوند.

### «قرارداد API» چیست؟
سند [`docs/API-CONTRACT.md`](./API-CONTRACT.md) فهرست همه‌ی APIهاست: هر آدرس چه می‌گیرد، چه برمی‌گرداند، و از کدام منبع (CMS یا NestJS) می‌آید. این سند **منبع حقیقت** است.

> **قانون طلایی:** هر تغییر در شکل یک API، **اول** در `API-CONTRACT.md` و `packages/shared` ثبت می‌شود، **بعد** در کد. هیچ‌وقت برعکس.

## ۶) راه‌اندازی روی سیستم خودت (یک‌بار)

### پیش‌نیازها
- **Node.js نسخه ۲۰** (از [nodejs.org](https://nodejs.org) نسخه LTS)
- **Git**
- **pnpm** — روش نصب در بخش بعد (مخصوص ویندوز)

### نصب pnpm (مخصوص ویندوز)
در PowerShell:
```powershell
corepack enable
corepack prepare pnpm@9.6.0 --activate
```
بعد **ترمینال را ببند و دوباره باز کن**، سپس چک کن:
```powershell
pnpm -v        # باید 9.6.0 بدهد
```
> اگر `not found` داد: یعنی ترمینال را ری‌استارت نکردی. حتماً پنجره را کامل ببند و باز کن.
> راه جایگزین: `npm install -g pnpm@9` بعد ری‌استارت ترمینال.

### گرفتن پروژه و اجرا
```bash
git clone https://github.com/Bigdaddy1935/roohbakhsh-website.git
cd roohbakhsh-website
pnpm install                                  # نصب همه‌ی پکیج‌ها
cp apps/web/.env.local.example apps/web/.env.local   # تنظیم آدرس API (در ویندوز: copy)
pnpm dev                                       # کل پروژه بالا می‌آید
```

بعد از `pnpm dev`:
- سایت → **http://localhost:3000/ar**
- API → **http://localhost:3001/api**
- مستندات API (Swagger) → **http://localhost:3001/api/docs**

## ۷) گردش کار روزانه (مهم‌ترین بخش)

همه روی **یک ریپوی مشترک** کار می‌کنند، ولی هر کس روی **برنچ خودش**. هیچ‌وقت مستقیم روی `main` کد نزن.

### چرخه‌ی هر کار
```
۱. صبح، آخرین کد را بگیر:
   git pull
   pnpm install          ← اگر package.json عوض شده بود، حتماً

۲. برای کارت یک برنچ بساز:
   git checkout -b feature/course-card

۳. با Claude Code کد بزن و تست کن (pnpm dev)

۴. قبل از تمام‌کردن، چک کن خطا نداشته باشی:
   pnpm typecheck
   pnpm lint

۵. کارت را بفرست بالا و PR بزن:
   git add -A
   git commit -m "توضیح کوتاه کار"
   git push -u origin feature/course-card
   → روی GitHub یک Pull Request باز کن

۶. Claude خودکار PR را ریویو می‌کند + ناظر تأیید می‌کند → merge روی main

۷. بقیه دوباره git pull می‌زنند و کد جدید را می‌گیرند
```

### ⚠️ دو قانون که نباید فراموش کنی
1. **هر `git pull` بعد از تغییر `package.json` → یک `pnpm install` لازم دارد.** وگرنه با پکیج قدیمی کار می‌کنی و خطا می‌خوری.
2. **روی `main` مستقیم کامیت نکن.** همیشه برنچ → PR.

## ۸) قوانین کدزنی (خلاصه‌ی `CLAUDE.md`)

- **تایپ مشترک:** همیشه از `@roohbakhsh/shared` استفاده کن. تایپ تکراری محلی نساز.
- **چندزبانگی:** هیچ متن قابل‌نمایشی را hard-code نکن. متن ثابت → فایل‌های `messages/`. متن داده‌ای → `Localized` با `value[locale]`.
- **RTL:** از خواص logical در CSS استفاده کن (`padding-inline-start` نه `padding-left`).
- **رنگ برند (۶۰-۳۰-۱۰):** فقط از متغیرهای رنگ استفاده کن، رنگ خام ننویس:
  - پس‌زمینه `#F8F9FA` · برند `#0CA789` · دکمه CTA `#FF9F1C` · متن `#2B2D42`
- **API:** خروجی هر endpoint باید از تایپ مشترک باشد؛ ورودی‌ها اعتبارسنجی شوند؛ خطا به شکل `ApiError`.

## ۹) نکته‌ی فنی: Next.js 16 و `params`

در Next.js نسخه ۱۵ به بعد، `params` در صفحه‌ها و لایوت‌ها **async (Promise)** شده. یعنی:

```tsx
// درست (Next 16):
export default async function Page({ params }: { params: Promise<{ locale: Locale }> }) {
  const { locale } = await params;
  ...
}
```
اگر صفحه یا لایوت جدید ساختی، حتماً همین الگو را رعایت کن.
