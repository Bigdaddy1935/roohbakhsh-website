# ریویوی روزانه‌ی کد با Claude Max (دستی، بدون هزینه‌ی API)

این روش برای **ناظر** است. به‌جای ریویوی خودکار روی گیت‌هاب، خودت با Claude Code روی سیستمت کد را بررسی می‌کنی. چون اشتراک **Max** داری، این کار **هزینه‌ی جداگانه ندارد**.

## آماده‌سازی (یک‌بار)

```bash
git clone https://github.com/Bigdaddy1935/roohbakhsh-website.git
cd roohbakhsh-website
```
و Claude Code را داشته باش (با همان اکانت Max لاگین).

## روال روزانه — ریویوی PRها

### ۱) ببین چه PRهایی باز است
```bash
gh pr list
```

### ۲) PR موردنظر را روی سیستمت بیاور
```bash
gh pr checkout <شماره‌ی PR>        # مثلاً: gh pr checkout 7
```

### ۳) داخل پروژه `claude` را باز کن و ریویو بگیر
دو راه:

**الف) دستور آماده:**
```
/review
```
این PR فعلی را ریویو می‌کند.

**ب) یا با زبان خودت بپرس** (دقیق‌تر، چون CLAUDE.md را هم لحاظ می‌کند):
```
این PR را طبق قوانین CLAUDE.md ریویو کن. روی این‌ها تمرکز کن:
باگ، مشکل امنیتی، استفاده‌نکردن از تایپ @roohbakhsh/shared،
متن hard-code شده، و رنگ خام به‌جای متغیر برند.
```

Claude کل diff را می‌خواند، قوانین پروژه را می‌خواند، و ایرادها را با شماره‌ی خط بهت می‌گوید.

### ۴) تصمیم بگیر
- اگر ایراد داشت → زیر PR کامنت بگذار تا برنامه‌نویس درست کند:
  ```bash
  gh pr comment <شماره> --body "متن ایراد..."
  ```
- اگر خوب بود → تأیید و merge:
  ```bash
  gh pr review <شماره> --approve
  gh pr merge <شماره> --squash
  ```

## روش جایگزین — مرور تغییرات یک روز

اگر نخواهی تک‌تک PRها را ببینی و فقط بخواهی «از دیروز تا حالا چه عوض شده»:
```bash
git fetch
git log --oneline --since="1 day ago" --all
git diff HEAD@{1} HEAD            # یا diff بین دو کامیت مشخص
```
بعد در Claude بپرس: «این تغییرات را برایم خلاصه و بررسی کن.»

## نکته
- چون محافظت `main` فعال است، هیچ PRی بدون **تأیید تو** merge نمی‌شود. پس حتی اگر یک روز ریویو نکنی، چیزی بدون نظرت وارد `main` نمی‌شود.
- اگر بعداً خواستی ریویوی خودکار را برگردانی، فقط کافی است در کنسول اعتبار API بگذاری و ورک‌فلوها را برگردانی (در تاریخچه‌ی گیت هست).
